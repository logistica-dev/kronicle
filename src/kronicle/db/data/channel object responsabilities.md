# Responsibility Matrix
| Layer / Class          | Responsibility                                                               | Connection / Transaction                                             | Notes                                                                                                                                               |
| ---------------------- | ---------------------------------------------------------------------------- | -------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| **ChannelDataManager** | Infrastructure layer: manages DB pool/connection, JSONB codec, transactions  | Owns connection / pool, provides `transaction()` context             | No domain knowledge; provides safe access to the DB for repositories and services                                                                   |
| **ChannelRepository**  | Persistence layer: CRUD for `ChannelResource` and its metadata/timeseries    | Accepts `conn` from DataManager; does **not** open/close connections | Constructs `ChannelResource` shells; orchestrates DB insert/fetch/delete; delegates row operations to `ChannelResource`                             |
| **ChannelResource**    | Aggregate root / domain model: holds `ChannelMetadata` + `ChannelTimeseries` | Operates with a `conn` passed from repository/service                | Encapsulates row validation, insertion, fetching, clearing; does **not** manage DB lifecycle; always represents a fully-initialized channel         |
| **ChannelTimeseries**  | Timeseries-specific behavior: rows storage, conversion, validation           | Operates with `conn` passed from `ChannelResource`                   | Responsible for transforming rows into DB tuples, verifying schema, clearing rows; completely unaware of metadata management or DB connection setup |
| **ChannelMetadata**    | Lightweight metadata storage                                                 | Operates with `conn` passed from repository                          | CRUD-only; no timeseries logic; upsert/fetch/delete metadata                                                                                        |

# Inputs to ChannelRepository
| Operation                    | Input                                                                  | Output                                                                                                   |
| ---------------------------- | ---------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------- |
| **Fetch / Get**              | `channel_id: UUID` (or other identifying keys)                         | `ChannelResource` shell populated from DB (`ChannelMetadata` + optionally pre-fetched `Timeseries` rows) |
| **Insert / Upsert Metadata** | `ChannelMetadata` (or a factory payload like `InputPayload`)           | `ChannelResource` shell (empty timeseries)                                                               |
| **Insert Rows**              | `ChannelResource` (already loaded or created) + rows                   | Rows inserted in DB; Resourceâ€™s in-memory rows may be cleared or left intact                             |
| **Delete Channel**           | `channel_id: UUID`                                                     | None (just deletes metadata + timeseries table)                                                          |
| **Fetch Rows**               | `ChannelResource` + optional filters (`from_date`, `to_date`, `limit`) | List of validated row dicts                                                                              |
